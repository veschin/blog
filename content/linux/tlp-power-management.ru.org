* Введение

На macOS и Windows производитель контролирует прошивку, ядро и железо совместно, поэтому энергосбережение работает без настройки. На Linux каждая подсистема (CPU, GPU, USB, WiFi, звук, диск, PCIe) независима; координатора по умолчанию нет, и ноутбук ведёт себя как десктоп. TLP[fn:1] заполняет роль такого координатора.

| Конфигурация        | Ёмкость | Потребление | Автономность |
|---------------------+---------+-------------+--------------|
| MacBook Air M2      | 52 Вт·ч | 5-7 Вт      | 15-18 ч      |
| Linux без настройки | 84 Вт·ч | 15-20 Вт    | 3-4 ч        |
| Linux + TLP         | 84 Вт·ч | 10-11 Вт    | ~6 ч         |

* Предпосылки

Шесть подсистем независимо влияют на потребление; каждую требуется настроить отдельно.

- CPU: частота, буст, энергопредпочтение (EPP)
- GPU: уровень производительности встроенной графики
- PCIe: режимы энергосна шины (ASPM)
- WiFi: агрессивность энергосна адаптера
- USB: автосуспенд (периодическое отключение питания устройства) портов
- Звук и диск: таймауты простоя

Инструменты управления питанием конфликтуют между собой; одновременно должен работать только один.

| Инструмент         | Год  | Охват                 | Конфликтует с         |
|--------------------+------+-----------------------+-----------------------|
| laptop-mode-tools  | 2005 | диск, CPU, USB        | TLP                   |
| TLP[fn:1]          | 2011 | всё + пороги заряда   | PPD, auto-cpufreq     |
| auto-cpufreq[fn:2] | 2020 | только CPU            | TLP                   |
| PPD[fn:3]          | 2021 | 4 параметра CPU       | TLP                   |
| tuned[fn:4]        | н/д  | CPU + I/O             | TLP                   |

PPD (Power Profiles Daemon, демон профилей питания) установлен по умолчанию в Ubuntu 22.04+, Fedora 36-40 и GNOME/KDE; при его наличии TLP не запустится. TLP 1.9+ включает компонент tlp-pd[fn:5], реализующий D-Bus API PPD, поэтому переключатели профилей рабочего стола продолжают работать поверх TLP. thermald и intel-lpmd с TLP не конфликтуют[fn:6].

* Установка

Перед установкой TLP необходимо удалить или заблокировать конкурирующие демоны; в противном случае PPD заглушит TLP через конфликт systemd без каких-либо сообщений об ошибках[fn:7].

Архитектурная зависимость: во всех дистрибутивах требуется заблокировать systemd-rfkill (службу управления радиомодулями), иначе она перехватит управление WiFi и Bluetooth у TLP.

#+begin_src bash
# Arch Linux
sudo pacman -S tlp tlp-rdw
sudo systemctl enable --now tlp.service
sudo systemctl mask systemd-rfkill.service systemd-rfkill.socket
#+end_src

#+begin_src bash
# Ubuntu / Debian (PPA автоматически удаляет PPD)
sudo add-apt-repository ppa:linrunner/tlp
sudo apt update && sudo apt install tlp tlp-rdw
sudo systemctl enable --now tlp.service
sudo systemctl mask systemd-rfkill.service systemd-rfkill.socket
#+end_src

#+begin_src bash
# Fedora 41+ (tuned заменил PPD как дефолт)
sudo dnf remove tuned tuned-ppd
sudo dnf install tlp tlp-rdw
sudo systemctl enable --now tlp.service
sudo systemctl mask systemd-rfkill.service systemd-rfkill.socket
# Fedora 36-40: заменить первую строку на:
# sudo dnf remove power-profiles-daemon
#+end_src

#+begin_src nix
# NixOS - встроенные assertions блокируют одновременное включение TLP и PPD
{ services.tlp.enable = true; }
#+end_src

* Настройка

** CPU

Два независимых механизма управляют частотой и тепловым бюджетом процессора.

Platform profile (профиль платформы, уровень прошивки, с ядра 5.12)[fn:8] контролирует тепловые лимиты, кривую вентилятора и пороги троттлинга. EPP (Energy Performance Preference, подсказка процессору об агрессивности экономии)[fn:9] действует на уровне планировщика частоты.

| Параметр                | Значения                             | Поддержка         |
|-------------------------+--------------------------------------+-------------------|
| Platform profile        | low-power / balanced / performance   | Lenovo, HP, ASUS, Dell, Framework |
| EPP                     | power / balance_power / balance_performance / performance | AMD Zen 2+ (ядро 6.5+), Intel Skylake+ |

На процессорах AMD оба механизма затрагивают одни регистры, поэтому их одновременное использование приводит к непредсказуемым результатам; эмпирически наблюдается залипание частоты на 1.3 ГГц. На процессорах Intel конфликта нет[fn:10].

#+begin_src conf
# /etc/tlp.conf - CPU, AMD-ноутбук
PLATFORM_PROFILE_ON_BAT=low-power
# EPP оставить пустым; заполнение вместе с platform profile = конфликт на AMD
CPU_ENERGY_PERF_POLICY_ON_BAT=
CPU_BOOST_ON_BAT=0
SCHED_POWERSAVE_ON_BAT=1
#+end_src

Отключение буста снижает многопоточную производительность примерно на 35%[fn:11], температуру с 90 до 55°C и увеличивает автономность на 30%; задачи, не требующие пиковой нагрузки, работают на базовой частоте 3-4 ГГц без заметной разницы.

** GPU

Встроенная графика AMD управляется через параметр DPM (Dynamic Power Management, динамическое управление питанием GPU)[fn:12].

| Параметр                   | Значения                | Описание                       |
|----------------------------+-------------------------+--------------------------------|
| RADEON_DPM_STATE_ON_BAT    | battery / balanced / performance | Уровень производительности GPU |

#+begin_src conf
RADEON_DPM_STATE_ON_BAT=battery
#+end_src

** PCIe

ASPM (Active State Power Management, энергосбережение шины PCIe)[fn:13] переводит NVMe и WiFi-адаптер в глубокий сон в простое.

| Режим          | Глубина сна | Риск                                 |
|----------------+-------------+--------------------------------------|
| default        | минимальная | нет                                  |
| powersave      | L1.2        | нет на большинстве устройств         |
| powersupersave | L1.2 sub-states | зависание NVMe Samsung, Xid 79 NVIDIA |

#+begin_src conf
PCIE_ASPM_ON_BAT=powersave
#+end_src

Режим powersupersave провоцирует зависание дисков Samsung при двойном входе в сон и ошибку Xid 79 ("GPU fallen off bus") на картах NVIDIA; подробности в разделе Грабли.

** WiFi

TLP выставляет один из двух уровней энергосбережения адаптера; второй уровень (у чипов Intel) управляется параметрами модуля ядра[fn:14].

#+begin_src conf
WIFI_PWR_ON_BAT=on
#+end_src

** USB и периферия

USB_AUTOSUSPEND переводит устройства в режим сна после истечения таймаута; игровые мыши, некоторые веб-камеры и DAC на нестандартных контроллерах при этом зависают или теряют синхронизацию.

#+begin_src conf
USB_AUTOSUSPEND=1
# Исключить устройства по VID:PID (заменить на реальные идентификаторы)
USB_DENYLIST="1234:5678"
#+end_src

Параметр назывался USB_BLACKLIST до TLP 1.4; старое имя молча игнорируется[fn:15].

** Звук

Кодек уходит в сон через заданное время тишины; при воспроизведении слышен щелчок при выходе из сна.

| Параметр                  | Рекомендуемое значение | Эффект                              |
|---------------------------+------------------------+-------------------------------------|
| SOUND_POWER_SAVE_ON_BAT   | 1                      | сон через 1 с тишины                |
| SOUND_POWER_SAVE_CONTROLLER| N                     | сон только кодека, не контроллера   |

#+begin_src conf
SOUND_POWER_SAVE_ON_BAT=1
SOUND_POWER_SAVE_CONTROLLER=N
#+end_src

Щелчок при значении 1 задокументирован в документации ядра[fn:16] как ожидаемое поведение; при значении 10 пауза перед засыпанием снижает частоту щелчков ценой ~0.1 Вт.

** Пороги заряда

Поведение сильно зависит от производителя; неподдерживаемые значения принимаются без ошибки, но молча игнорируются.

| Производитель | Диапазон        | Поддержка ядра | Примечание                          |
|---------------+-----------------+----------------+-------------------------------------|
| ThinkPad      | 0-99 / 1-100%   | 5.17+          | полный контроль                     |
| IdeaPad/Yoga  | ~60% (фикс.)    | н/д            | только режим conservation on/off    |
| ASUS          | 1-100%          | н/д            | многие модели принимают 40/60/80    |
| Dell          | 50-95 / 55-100% | 6.12+          | минимальная разница 5%              |
| Framework     | 0-99 / 1-100%   | 6.12+          | EC-прошивки свежее 3.05 ломают[fn:17], исправлено в ядре 6.17 |
| Samsung       | 80% (фикс.)     | н/д            | бинарный тоггл                      |
| Toshiba       | 80/100%         | н/д            | сохраняется в NVRAM                 |

#+begin_src conf
# Пример для ThinkPad
START_CHARGE_THRESH_BAT0=75
STOP_CHARGE_THRESH_BAT0=80
#+end_src

* Грабли

Каждый симптом из таблицы проявляется независимо и требует отдельного вмешательства; применение всех фиксов сразу безопасно.

| Подсистема    | Симптом                                        | Причина                              | Фикс                                                  |
|---------------+------------------------------------------------+--------------------------------------+-------------------------------------------------------|
| Bluetooth     | мышь зависает, пропуск клавиш, обрыв аудио    | btusb автосуспенд в ядре             | =options btusb enable_autosuspend=0= в /etc/modprobe.d/ |
| WiFi Intel    | задержка 60 мс, разрывы, таймауты DNS         | TLP не покрывает второй уровень PWM  | =iwlwifi power_save=0= + =iwlmvm power_scheme=1=      |
| WiFi MediaTek | разрыв раз в 11-12 ч                          | регрессия драйвера mt76              | обновление до ядра 6.19                               |
| WiFi Qualcomm | деградация при подключённом BT                | конфликт RF-цепочек                  | обновление прошивки адаптера                          |
| PCIe ASPM     | зависание NVMe Samsung; Xid 79 NVIDIA         | powersupersave активирует L1.2 sub-states | =PCIE_ASPM_ON_BAT=powersave=                     |
| Звук          | щелчок при воспроизведении                    | кодек выходит из сна                 | =SOUND_POWER_SAVE_ON_BAT=10=                          |
| Диск WD       | ритмичный щелчок каждые 5-30 с               | WD паркует головки при любом APM 1-254 | =DISK_APM_LEVEL_ON_BAT="255"=                       |
| USB DAC/мышь  | зависание, рассинхронизация                   | автосуспенд на нестандартном контроллере | VID:PID в USB_DENYLIST                             |
| GPU AMD       | мерцание OLED-панели                          | PSR (самообновление панели) с ядра 5.16 | =amdgpu.dcdebugmask=0x10= в параметрах ядра         |
| Suspend       | WiFi "подключён", пакеты не идут; BT меняет номер hci | правила TLP не переприменяются | systemd-сервис на resume                              |

** Bluetooth

Параметр USB_EXCLUDE_BTUSB[fn:18] не покрывает все контроллеры; единственный надёжный способ отключить автосуспенд Bluetooth состоит в настройке параметра модуля ядра.

#+begin_src conf
# /etc/modprobe.d/btusb.conf
options btusb enable_autosuspend=0
#+end_src

** WiFi Intel

TLP контролирует только один уровень из двух для чипов Intel; второй требует явной настройки модуля ядра[fn:19].

#+begin_src conf
# /etc/modprobe.d/iwlwifi.conf
options iwlwifi power_save=0
options iwlmvm power_scheme=1
#+end_src

** USB

Идентификаторы VID:PID отображаются через =lsusb=; устройства с нестандартным firmware (игровые мыши, некоторые DAC) не входят в стандартный список исключений TLP.

** GPU AMD (мерцание)

PSR (Panel Self Refresh, технология обновления только изменившихся участков экрана)[fn:20] включён по умолчанию с ядра 5.16 и провоцирует мерцание на OLED-панелях.

#+begin_src conf
# /etc/kernel/cmdline или GRUB_CMDLINE_LINUX
amdgpu.dcdebugmask=0x10
#+end_src

** Suspend/resume

После выхода из сна правила TLP не переприменяются автоматически; WiFi-адаптер сохраняет состояние "подключён", но не пропускает пакеты.

#+begin_src systemd
# /etc/systemd/system/tlp-resume.service
[Unit]
Description=TLP resume reapply
After=suspend.target hibernate.target hybrid-sleep.target

[Service]
Type=oneshot
ExecStart=/usr/sbin/tlp resume

[Install]
WantedBy=suspend.target hibernate.target hybrid-sleep.target
#+end_src

#+begin_src bash
sudo systemctl enable tlp-resume.service
#+end_src

* Проверка

| Команда                    | Что показывает                              |
|----------------------------+---------------------------------------------|
| =tlp-stat -s=              | статус службы, активный профиль            |
| =tlp-stat -p=              | текущие параметры CPU, platform profile     |
| =tlp-stat -b=              | состояние батареи, пороги заряда            |
| =tlp-stat -u=              | USB-устройства, статус автосуспенда         |
| =powertop --html=/tmp/p.html= | разбивка потребления по устройствам      |
| =turbostat --Summary=      | частоты, C-states, потребление пакета CPU   |
| =s-tui=                    | интерактивный мониторинг частоты и температуры в терминале |
| =cat /sys/class/power_supply/BAT0/power_now= | мгновенное потребление в мкВт |

powertop с флагом =--auto-tune= конфликтует с TLP[fn:21]: при одновременной работе параметры перезаписываются непредсказуемо; флаг допустим только для разовой диагностики без запущенного TLP.

* Результат

Замеры выполнены на Lenovo XiaoXinPro 14, Arch Linux 6.19.12, Hyprland 0.54; нагрузка: браузер + терминал + IDE, яркость 30%, WiFi включён.

| Параметр             | До настройки    | После TLP        |
|----------------------+-----------------+------------------|
| Потребление          | 15-20 Вт        | 10.7 Вт          |
| Автономность         | 3-4 ч           | ~6 ч             |
| Частота в простое    | 2.5-3 ГГц       | 1.1-1.4 ГГц      |
| Частота под нагрузкой | до 5.1 ГГц     | 3.0-3.2 ГГц      |
| Температура пакета   | 70-90°C         | 45-55°C          |

Итоговая конфигурация:

#+begin_src conf
# /etc/tlp.conf - агрессивный режим, AMD-ноутбук
PLATFORM_PROFILE_ON_BAT=low-power
CPU_ENERGY_PERF_POLICY_ON_BAT=
CPU_BOOST_ON_BAT=0
SCHED_POWERSAVE_ON_BAT=1
PCIE_ASPM_ON_BAT=powersave
RADEON_DPM_STATE_ON_BAT=battery
USB_AUTOSUSPEND=1
WIFI_PWR_ON_BAT=on
SOUND_POWER_SAVE_ON_BAT=1
SOUND_POWER_SAVE_CONTROLLER=N
#+end_src

Разница в автономности по сравнению с нетронутой системой составляет 1.5-2× при сохранении комфортной рабочей частоты и без деградации в задачах, не требующих буста.
