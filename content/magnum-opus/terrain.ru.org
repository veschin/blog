* Визуальный стиль

Визуальная основа проекта --- статья David Holland [[https://www.davidhol.land/articles/3d-pixel-art-rendering/][3D Pixel Art Rendering]]. В ней описан набор из 11 приёмов для превращения трёхмерной сцены в пиксель-арт --- стилизацию под пиксельную графику.

#+CAPTION: Референс: meadow scene из статьи David Holland
[[file:/magnum-opus/ref-holland-meadow.png]]

Три ключевых приёма, перенесённых в проект:

- *Столбцовый рендеринг.* Каждая ячейка карты --- вертикальный столбец. Высота столбца соответствует высоте ландшафта в данной точке. Результат напоминает воксельную[fn:voxel] графику, но реализован на обычных 3D-примитивах.
- *Toon-шейдинг.* Материал с дискретными уровнями освещения. Фотореалистичные градиенты заменяются резкими цветовыми переходами --- стилистика комиксов и мультфильмов.
- *Ортографическая камера.* Камера без перспективных искажений: параллельные линии остаются параллельными независимо от расстояния. Усиливает стилизацию, характерную для пиксель-арта.

Стек: Rust, Bevy 0.18[fn:bevy].

[fn:voxel] Воксель --- объёмный пиксель, элементарный куб в трёхмерном пространстве. Minecraft --- наиболее известный пример воксельной графики.
[fn:bevy] Bevy --- фреймворк для создания игр на Rust. Использует архитектуру ECS (Entity-Component-System), где данные хранятся отдельно от логики.

* Первый рендер: плоский мир

#+CAPTION: v1: плоский грид 32×32, первые примитивы на поверхности
[[file:/magnum-opus/01-first-render.png]]

Стартовая точка --- плоская сетка 32×32 ячейки. Ни высот, ни воды, ни ресурсов. Серые кубики на зелёной плоскости.

Основная структура данных --- плоская сетка, сохранившаяся без изменений через все девять версий генератора:

#+begin_src rust
pub struct FlatGrid<T> {
    pub width: usize,
    pub height: usize,
    pub data: Vec<T>,
}

impl<T: Copy + Default> FlatGrid<T> {
    pub fn new(width: usize, height: usize) -> Self {
        Self {
            width, height,
            data: vec![T::default(); width * height],
        }
    }

    pub fn get(&self, x: i32, z: i32) -> T {
        self.data[z as usize * self.width + x as usize]
    }

    pub fn set(&mut self, x: i32, z: i32, val: T) {
        self.data[z as usize * self.width + x as usize] = val;
    }
}
#+end_src

Одномерный массив с двумерной адресацией. Обобщённый параметр позволяет хранить в одной структуре высоты (числа с плавающей точкой), типы ландшафта (перечисление), булевы маски и дистанции. Одна структура --- все слои данных генератора.

* Карта высот и вода: первые неудачи

#+CAPTION: v2-v3: карта высот, река, залежи ресурсов
[[file:/magnum-opus/02-terrain-tiles.png]]

В версиях v2-v3 появляется карта высот[fn:heightmap] на основе шума Перлина[fn:noise] --- 21 уровень от -5 до +5. Впервые возникают утёсы, озёра и перепады высот.

Проблема обнаруживается при тестировании на разных начальных числах генератора[fn:seed]. Параметр смещения шума контролирует долю суши: значение 1.5 порождает избыток океана, 2.0 исправляет один seed но ломает другой, 3.0 и 4.0 --- аналогично. Каждое значение работает локально и разрушается глобально.

*Вывод:* четыре итерации подбора, четыре разных seed-а сломаны. Значение, работающее на всех seed-ах одновременно, не найдено.

[fn:heightmap] Карта высот --- двумерная сетка, где каждой ячейке присвоено числовое значение высоты. Визуально: чем выше значение, тем выше рельеф.
[fn:noise] Шум Перлина --- алгоритм генерации псевдослучайного рельефа. Производит плавные, природоподобные перепады высот вместо хаотичного разброса.
[fn:seed] Seed (начальное число) --- число, определяющее конкретный вариант случайной генерации. Один seed всегда порождает одинаковый результат.

* Симуляция дождя: тупиковая ветка

Следующая попытка --- алгоритм D8 (drainage по восьми направлениям):
1. Каждая ячейка получает единицу осадков.
2. Вода стекает к соседу с наименьшей высотой.
3. Ячейки с накоплением выше порога образуют реки.
4. Локальные минимумы --- впадины без стока --- образуют озёра.

Результат: ни одной сухой ячейки на карте 64×32. Сплошной океан вместо рек и озёр.

*Причина:* D8 моделирует дренаж дождевой воды. На ограниченном острове внутренние ячейки дренируются к краям, накопление концентрируется на периферии. Порог образования рек зависит от размера мира --- устойчивого значения не существует.

*Ключевой вывод:* задача формулировалась как "где накапливается вода?" Продуктивная формулировка --- "где расположены источники?"

* Система источников: прорыв

Система источников инвертирует задачу. Наиболее вероятный сбой прежнего подхода --- неуправляемый избыток воды (неустраним). Наиболее вероятный сбой нового --- недостаток воды (устраним: добавить источники). Контролируемая плотность заменяет неуправляемую симуляцию.

Размещение источников определяется функцией оценки. Для каждой ячейки выше уровня моря вычисляется оценка: высота (с ограничением сверху) умноженная на расстояние от края мира. Ограничение высоты предотвращает уход всех источников на горные пики. Расстояние от края обеспечивает длинные русла.

#+begin_src rust
candidates.sort_by(|a, b| {
    let dist_a = a.0.min(ww - 1 - a.0).min(a.1.min(wh - 1 - a.1)) as f32;
    let dist_b = b.0.min(ww - 1 - b.0).min(b.1.min(wh - 1 - b.1)) as f32;
    let score_a = heights.get(a.0, a.1).min(3.0) * dist_a;
    let score_b = heights.get(b.0, b.1).min(3.0) * dist_b;
    score_b.partial_cmp(&score_a).unwrap()
});

let mut springs: Vec<(i32, i32)> = Vec::new();
for &c in &candidates {
    if springs.len() >= total_springs { break; }
    let too_close = springs.iter()
        .any(|s| (s.0 - c.0).abs().max((s.1 - c.1).abs()) < wc.spring_min_spacing);
    if !too_close {
        springs.push(c);
    }
}
#+end_src

Минимальное расстояние между источниками предотвращает кластеризацию. Два-три источника на тайл[fn:tile] достаточно для игровых гарантий.

[fn:tile] Тайл --- фрагмент мира фиксированного размера (32×32 ячейки). Мир составляется из нескольких тайлов.

** Трассировка рек

Каждый источник трассирует реку по градиенту высот --- от точки с большей высотой к точке с меньшей. Жадный спуск[fn:greedy] с 25% вероятностью отклонения формирует естественные изгибы русла:

#+begin_src rust
let chosen = if h_best < h_curr && valid.len() >= 2 {
    let jit = hash64(
        wc.spring_seed
            .wrapping_add(path.len() as u64 * 31)
            .wrapping_add(start.0 as u64 * 97 + start.1 as u64 * 13),
    );
    if jit % 100 < wc.meander_chance as u64 {
        let h_alt = if is_ocean.get(valid[1].0, valid[1].1) {
            ws.sea_level - 1.0
        } else {
            heights.get(valid[1].0, valid[1].1)
        };
        if h_alt <= h_curr + wc.meander_height_tolerance {
            valid[1] // второй по высоте сосед
        } else {
            best
        }
    } else {
        best
    }
} else {
    best
};
#+end_src

Детерминированный хеш[fn:hash] от позиции и длины пути заменяет генератор случайных чисел --- одно и то же начальное число всегда порождает одинаковый ландшафт.

[fn:greedy] Жадный алгоритм --- на каждом шаге выбирает локально лучший вариант (ближайший сосед с наименьшей высотой), не заглядывая вперёд.
[fn:hash] Хеш-функция --- преобразование входных данных в псевдослучайное число. Одинаковый вход всегда даёт одинаковый выход.

** Заполнение озёр

Когда русло упирается в подъём (следующая ячейка выше текущей), запускается заполнение поиском в ширину[fn:bfs]:

#+begin_src rust
if h_chosen > h_curr {
    let fill_level = if h_chosen - h_curr >= wc.lake_fill_margin {
        h_chosen + wc.lake_fill_margin
    } else {
        h_chosen
    };
    let mut lake_cells = Vec::new();
    let mut fill_queue = VecDeque::new();
    fill_queue.push_back(current);

    while let Some(cell) = fill_queue.pop_front() {
        if lake_cells.len() >= wc.lake_max_cells { break; }
        for (nx, nz) in [(cell.0+1,cell.1), (cell.0-1,cell.1),
                         (cell.0,cell.1+1), (cell.0,cell.1-1)] {
            if heights.get(nx, nz) <= fill_level
                && !is_water.get(nx, nz)
                && !lake_set.get(nx, nz)
            {
                lake_set.set(nx, nz, true);
                lake_cells.push((nx, nz));
                fill_queue.push_back((nx, nz));
            }
        }
    }
}
#+end_src

Ограничение в 25 ячеек предотвращает затопление мира. Менее двух затопленных ячеек --- лужа, не озеро; такой результат отбрасывается.

[fn:bfs] Поиск в ширину (BFS) --- обход соседних ячеек волной. Сначала проверяются непосредственные соседи, затем соседи соседей и т.д. Результат --- заполнение связной области.

* Масштабирование: от одного тайла к континенту

#+CAPTION: v7: мир из нескольких тайлов, океанские столбцы, утёсы по краям
[[file:/magnum-opus/03-tile-expansion.png]]

Мир составляет 256×256 ячеек (8×8 тайлов). Генерация глобальная --- единый проход шума по всей площади, швы между тайлами отсутствуют.

Три решения, определивших масштабируемость:

- *Океанские столбцы вместо плоскости.* Плоскость океана перекрывала внутренние долины. Океанские ячейки теперь отображаются как столбцы на уровне моря, фон --- сплошной цвет. Долины и впадины остаются видимыми.
- *Вырезание русел.* Реки углубляют ландшафт на полуровень, формируя видимые каналы с обрывистыми берегами.
- *Гарантия элементов.* Генератор обязан обеспечить наличие хотя бы одного озера и одного водопада (пересечение реки и обрыва). Отсутствие любого из них --- ошибка генерации.

* Штампы и ресурсы

#+CAPTION: v8: геометрические штампы, 15 материалов, залежи ресурсов
[[file:/magnum-opus/04-full-terrain.png]]

Версия v8 вводит явные геометрические штампы поверх шума: горы, впадины, рифты[fn:rift].

Первоначальная реализация использовала операцию максимума --- штамп применялся только если его высота превышала существующую. Отладочные метрики демонстрируют проблему: шум самостоятельно производит 364 ячейки выше порога, штамп через максимум добавляет ещё 8. Восемь из трёхсот шестидесяти четырёх --- визуально неотличимо от отсутствия штампа.

Исправление --- операция присвоения. Штампы записывают абсолютные значения, доминируя над шумом. Отладочные метрики оказываются точнее визуальной оценки.

Ширина рек увеличивается от 1 ячейки у истока до 3 у устья. Расширение выполняется перпендикулярно направлению течения.

Три типа ресурсов (медь, металл, уголь) размещаются по принципу золотого угла[fn:golden] с индивидуальными параметрами: количество кластеров, узлов в кластере, радиус.

[fn:rift] Рифт --- протяжённая трещина в ландшафте. Генерируется алгоритмом Брезенхема (построение прямой линии по сетке) с добавлением случайного отклонения.
[fn:golden] Золотой угол (~137.5°) --- угол, обеспечивающий максимально равномерное распределение точек по окружности. Применяется в ботанике (расположение семян подсолнуха) и компьютерной графике.

* Горы: плавный профиль

#+CAPTION: v9: горы высотой 15-40 единиц, 30 проходов сглаживания
[[file:/magnum-opus/05-mountains.png]]

Форма горы определяется двумя математическими приёмами.

*Гладкая интерполяция (smoothstep).* Функция, обеспечивающая плавный переход от единицы к нулю с нулевой производной на обоих концах --- вершина и основание горы переходят в окружающий ландшафт без видимого шва:

#+begin_src rust
let t = (dist / radius).clamp(0.0, 1.0);
let blend = 1.0 - t * t * (3.0 - 2.0 * t);

let jitter = noise_octave(nx, nz, mc.jitter_scale, jitter_seed) * mc.jitter_amp;
let target_h = (peak_h + jitter).clamp(0.0, peak_h);

let existing = heights.get(nx, nz);
let new_h = existing + (target_h - existing) * blend;
heights.set(nx, nz, new_h.max(existing));
#+end_src

Формула =1 - t²(3 - 2t)= возвращает единицу в центре и ноль на краю. Наложение шума (±2.0) разрушает геометрическую правильность конуса, формируя органические контуры.

*Сглаживание после квантования.* Квантование[fn:quant] преобразует плавные высоты в дискретные ступени --- визуальная ступенчатость. 30 проходов усреднения соседних ячеек устраняют ступенчатость, сохраняя общую форму. Критический порядок операций: сглаживание выполняется строго *после* квантования. Обратный порядок бессмыслен: квантование после сглаживания заново воссоздаёт ступени, которые только что устранялись.

Каждая гора уникальна: высота пика от 15 до 40 единиц, радиус основания от 12 до 25 ячеек. Иерархия материалов формирует визуальную высотность: трава -> камень (выше 3.0) -> снежная вершина (выше 6.0).

[fn:quant] Квантование --- округление непрерывных значений до дискретных уровней. Например, высота 2.37 округляется до 2.5. Создаёт характерный ступенчатый рельеф.
